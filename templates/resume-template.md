# レジュメ用テンプレ（完成・抽象化時の出力形式）

職務経歴書・ポートフォリオへ転用する**完成版**の形式。`appeals/` の進行中ドラフト（`appeal-template.md` 形式）を、タスク完了・レジュメ化のタイミングでこの形式へ変換し、**同時に固有名を抽象化**する（`CLAUDE.md`「固有名の扱い（具体↔抽象のライフサイクル）」）。

- 1プロジェクト＝**詳細版／職務経歴書向け圧縮版**の2バリアントを出す。各バリアントは同じ5見出し（課題 / 解決策 / 成果 / 関与度 / 使用技術）。
- **タイトルは体言止めにせず「〜を実装し、〜により〜を解消」と動詞で締める**一文サマリにする。
- **課題・成果には定量を必ず織り込む**（頻度×1件あたり＝合計工数、before→after 等）。実数が無い箇所はプレースホルダー（`◯◯` / `＜実データを記入＞`）にして明示する。
- 社名・パートナー名・人物名・社内固有名（システム/リポジトリ/チケット/内部実装名）は**抽象化**。一般技術名（Go / Firebase / Slack 等）は実名で残す。

### 解決策

### 成果

（課題が解消したこと。現象がどう無くなったか）

（**当事者性**を1〜2文で。自ら調査・判断・提案したこと＋どこからどこまで担当したか。例「制約を自ら調査し、◯◯の比較から△△を判断・提案。影響調査・設計・実装・テストまで一貫して担当」）
