# career-portfolio 強化アイデア

> 最終更新: 2026-06-27
> このファイルは「career-portfolio をどう強化していくか」のアイデアを貯める場所。
> 機能の実装記録ではなく、**着想 → 育成 → 実装候補**へ昇格させるためのアイデア箱。

## ビジョン

「業務経験の整理ハブ」から、**転職活動の上流（やりたいこと／なりたい姿／企業選び）から下流（レジュメ・面接）までを一気通貫で伴走する個人ツール**へ育てる。
精度向上の仕掛けとして、**役割の異なるサブエージェント同士のディスカッション**（ドラフト→批判→統合 等）を取り入れる。

## 使い方（Inbox → Backlog の昇格運用）

1. 思いついたら、まず `📥 Inbox` に一行で気軽に追記する（分類しない）。
2. 育ってきた（やる価値・形が見えてきた）アイデアを `📋 Backlog` へ昇格し、構造化する。
3. Backlog の各項目は、応募先や優先度に応じて実装候補として選んでいく。

## 🧭 使い方の流れ（シナリオ例）

**B1〜B18 の番号は「実行順」ではなく、フェーズで並べた整理番号**。実際は状況・きっかけに応じて、必要な部品を必要な順で繋ぐ。同じ部品（例: B4 強み発掘）が複数の流れに何度も登場する＝直線の手順ではなくネットワーク。

- **起点①「気になる求人を見つけた」**: `B13 JDマッチング → B12 企業分析&受験判定 → （受けると決めたら）B11 職務経歴書の自動組み立て →（仕上げ）B15 模擬面接`
- **起点②「経験が溜まってきた・棚卸ししたい」**: `B1 自動棚卸し → B4 強み発掘 → B3 スキル集計 → B7 レジュメ・ブラッシュ`（応募先は未定でOK）
- **起点③「そもそも何がやりたいか分からない」**: `B5 キャリア言語化 → B12 入るべき企業の言語化・判定 → B13 具体の求人とマッチング`
- **起点④「面接が近い」**: `B15 模擬面接 → B16 Q&A蓄積 →（弱点が見えたら）B6 自己PR / B10 職経 を補強`

## 🤝 サブエージェント設計メモ

役割の異なるエージェントを用意し、ディスカッションさせて精度を上げる。Backlog の各アイデアはこの役割を組み合わせて実現する。

### 役割カタログ（案）

| 役割 | 観点・仕事 |
|---|---|
| 🔍 深掘りインタビュアー | 経験・価値観・判断の背景を質問で引き出す。当事者性のエピソードを掘る |
| 😈 批判役（Devil's Advocate） | 弱い主張・盛り過ぎ・抽象的な箇所を突く。反証を試みる |
| 🧑‍💼 採用担当役 | 企業の採用視点で「響くか/懸念点」を指摘する |
| ✍️ 編集役 | 簡潔で伝わる日本語へ整える（冗長・曖昧の除去） |
| 📊 事実・定量検証役 | 数値の裏付けを確認し、根拠の無い数値はプレースホルダー化する |
| 🧭 キャリアコーチ役 | やりたいこと/なりたい姿/価値観の言語化を伴走する |
| 🏢 企業分析役 | 企業研究（事業・カルチャー・将来性）と、キャリア整合性の分析 |
| ⚖️ ジャッジ（統合役） | 複数の意見を統合し、結論と次アクションを出す |

### 討議パターン（案）

1. **ドラフト → 批判 → 反映 → ジャッジ**: 1案を作り、批判役が突き、反映し、ジャッジが確定。
2. **パネルレビュー**: 複数の役割が同時にレビューし、ジャッジが統合（取捨選択の根拠も残す）。
3. **模擬面接**: 面接官役 ↔ 回答者、観察役がフィードバック。出力は B16（Q&A蓄積）へ。

### 実現の足場（メモ）

- この環境の **サブエージェント（Agent ツール）** で役割別エージェントを起動できる。
- **Workflow オーケストレーション** で、役割別エージェントの並列実行・討議（ドラフト→批判→統合）を組める。
- まずは軽量に「1ドラフト＋1批判＋1統合」から始め、効果を見て役割数・ラウンド数を増やす。
