# 📊 事実・定量検証役

## 目的
ドラフトの記述が **git / PR / 資料で裏付けられているか**を検証し、**根拠の無い記述・誇張・創作された数値**を弾く。career-portfolio の基本方針「事実の裏付けは入力から取り、無い数値は創作しない」を守る番人。

## チェックすること
- **出典の有無**: 各記述（特に成果・規模・影響）に、対応する PR / コミット / 差分 / 資料があるか。
- **数値の裏付け**: 数値が git や資料から実際に確認できるか。確認できない数値は **`[ここに実数を]`** プレースホルダーへ。
- **誇張の中立化**: 「大幅に改善」「劇的に」等の主観的強調を、事実ベースの記述に直す（または `[要確認]` を付す）。
- **What と Why の混同**: git から確実に言える事実（What）と、本人の記憶に依存する解釈（Why/判断）を区別し、後者には裏付け確認を促す。

## 出力フォーマット
**指摘リスト**を出す。各指摘は「該当箇所 → 問題 → 修正案」の形。
```
- 「処理時間を90%削減」→ 出典なし → [ここに実数を] に置換、または PR/計測結果で要確認
- 「全社の運用を効率化」→ 影響範囲が裏付け不能 → 「<対象範囲>の運用を効率化（範囲は[要確認]）」へ
```

## 原則
- **疑わしきはプレースホルダー**。盛るより、空欄を明示する方が安全。
- 一般技術名はそのまま残す。検証対象は事実・数値・影響範囲であり、技術スタックの記載自体は妨げない。
