# B1. git履歴/PR からの自動棚卸し（auto-inventory）

実装してきた機能を git/PR から掘り起こし、**アピールできそうなものを選別 → 深掘りして `sources/` の素材にする**ツール。職務経歴書の作成・ブラッシュアップ時に「これまで何をやったか」を漏れなく振り返るために使う。

- **対応ゴール**: ①経験からの強み発見（の入口） / 棚卸し・素材づくり
- **出口（スコープ）**: `sources/<会社>/` の素材（深掘りメモ）まで。**appeal 文は作らない**（それは B6 等の下流ツールの仕事）。
- **使うサブエージェント**: [[interviewer]]（深掘り質問）/ [[verifier]]（裏付け検証）

## 起動文（コピペして使う）

**Stage 1（棚卸し一覧をつくる）**:
```
@career-portfolio/tools/b1-auto-inventory/playbook.md に従って、
@<対象リポジトリ> の <期間（例: 直近3ヶ月 / 2026-01以降）> を棚卸しして
```

**Stage 2（候補を深掘りして素材化）**:
```
@career-portfolio/tools/b1-auto-inventory/playbook.md に従って、
@<対象リポジトリ> の PR #<番号> を深掘りして
```

## 入出力

| | Stage 1（sweep） | Stage 2（deep-dive） |
|---|---|---|
| 入力 | 対象リポジトリ + 期間 | 対象リポジトリ + PR番号 |
| 処理 | PR を機能単位に抽出し暫定スコア付け | PR/差分から自動ドラフト → 質問生成 → 検証 |
| 出力 | 台帳 `sources/<会社>/_inventory.md`（蓄積） | 素材 `sources/<会社>/<テーマ>.md`（下書き） |

## 前提・原則

- **git は読むだけ**。`git add/commit/push` はしない（career-portfolio・対象リポジトリ双方とも）。
- 事実は git/PR から取り、**無い数値・判断理由は創作せずプレースホルダー化**。
- 社内固有名は**実名で記録**（抽象化はレジュメ化時に一括）。一般技術名はそのまま。
- `gh`（GitHub CLI）が使える前提。使えない場合は `git log` ベースにフォールバック（playbook 参照）。
