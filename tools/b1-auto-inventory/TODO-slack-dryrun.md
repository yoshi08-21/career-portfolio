## 1. 検証項目（うまく動くか）

### フェーズA：アンカー探索（自分起点で探せるか）
- [ ] **チケットID検索**が該当スレッドに当たるか（`STELLA-7279`）。**表記揺れ**（`STLLA-`, スペース有無, 小文字）でも拾えるか／追加クエリが要るか。
- [ ] **PR URL / デプロイURL 検索**（`pr-1928-...`）でスレッドに当たるか。
- [ ] **`filter_users_from=<本人>`** で自分の発言だけに絞れるか（本人IDの解決が正しいか）。
- [ ] **メンション検索**（`<@本人ID>`）が機能するか。表記揺れでの取りこぼし度合い。
- [ ] ノイズ（無関係ヒット）の割合。多すぎないか＝クエリを絞る必要があるか。

### フェーズB：スレッド取得（流れを把握できるか）
- [ ] 検索結果の `Channel` + `ThreadTs` を `conversations_replies` に渡してスレッド全体が取れるか。
- [ ] **スレッドでない単発メッセージ**（`thread_ts` がメッセージ自身）の扱いが破綻しないか。
- [ ] `limit`（既定 `1d`）で十分か。長いスレッドでページング（cursor）が要るか。

### ① 索引（dedup が効くか）
- [ ] 2回目に同じスレッドへ来たとき、`_slack-threads.local.md` 照合で**再 `conversations_replies` をスキップ**できるか。
- [ ] 状態（`確認済`/`部分`/`無関係`）の運用が実際に回るか。`無関係` で次回スキップできるか。

### ② アーカイブ（粒度・マスク品質）
- [ ] **自分の発言＝原文寄り**で残せているか。
- [ ] **他者の発言＝要約だが議論の流れが保たれているか**（論点・懸念・合意の方向が消えていないか／削りすぎていないか）。
- [ ] **個人情報マスク**の取りこぼしが無いか（氏名・役職での暗黙特定・取引先名・連絡先）。
- [ ] **秘密情報の完全除外**ができているか（トークン/鍵/`.env`/URL内資格情報を書いていないか）。
- [ ] **用語対応表**が埋まり、後で一括置換できる形か。
- [ ] 逐語ダンプになっていないか（中精度に収まっているか）。

### ③ 素材（appeal 元ネタとして使えるか）
- [ ] `背景／工夫・判断／当事者性メモ` に Slack 由来の文脈が反映され、**git だけのときより当事者性が厚く**なったか。
- [ ] 裏取りできない数値が**プレースホルダーのまま**か（創作していないか）。
- [ ] ③（追跡対象）に生文・permalink・実名が漏れていないか。

### ガードレール（安全）

- [ ] `git add/commit/push` を**していない**（提示のみ）。
