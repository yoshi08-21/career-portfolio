# career-portfolio

業務で得た経験を、転職活動で使える**アピール文（職務経歴書・ポートフォリオ・面接素材）**へ整理・蓄積するための個人リポジトリ。**特定の会社・プロジェクトに依存しない横断ハブ**として運用し、レジュメを作りたいプロジェクトのディレクトリをメンションして、その経験の整理を依頼する。

目的は「**経験 → 整理 → アピール → レジュメ転記**」をつなぐこと。

## 配置と前提

複数プロジェクト（在籍企業）の**親ディレクトリ**にこのリポジトリを置く（独立した git リポジトリ）。横断ルートでセッションを開けば、career-portfolio と各プロジェクトを短い相対パスでメンションできる。

```text
~/Applications/development/            ← 横断ルート（ここでセッションを開くのが推奨）
├── career-portfolio/                  ← このリポジトリ
├── supernova/ …                       ← 対象プロジェクト（兄弟）
└── <other-projects>/                  ← 他プロジェクトも兄弟に並ぶ
```

## 基本の流れ

```text
対象プロジェクトの入力（資料メンション／git履歴／メモ 等）+ 自分の粗いメモ
        ↓ （AI が整理）
アピール文（課題 → 施策 → 工夫 → 成果）＋ アピールポイント候補を5軸で評価してストック
        ↓ （レジュメ転記時：応募先に合わせAIが上位2〜3点を選定）
レジュメ完成版（課題/解決策/成果/関与度/使用技術）に整形・固有名を抽象化
        ↓
職務経歴書・ポートフォリオ・面接で使う
```

## ディレクトリ構成

経験は**会社／在籍コンテキスト単位**でディレクトリを切って整理する。

| パス | 役割 |
|---|---|
| `sources/<会社>/` | 経験の素材置き場。粗い箇条書きメモ・関連資料へのリンクなど、整理前のインプット |
| `appeals/<会社>/` | 完成・下書きのアピール文（成果物）。1テーマ＝1ファイル（`YYYY-MM-<テーマ>.md`） |
| `templates/` | 進行中ドラフト用テンプレ（`appeal-template.md`）・レジュメ完成版テンプレ（`resume-template.md`＝課題/解決策/成果/関与度/使用技術 × 詳細版/圧縮版）・依頼テンプレ（`request-template.md`）・選択依頼テンプレ（`selection-request-template.md`） |
| `resume/` | （任意・必要になったら追加）職務経歴書・履歴書本体 |
| `CLAUDE.md` | AI に整理を頼むときの方針・入力チェックリスト・出力フォーマット・選択ルーブリック |

## 使い方（1テーマを追加するとき）

別セッションで始めるときの開始ルールは **`START-HERE.md`** にまとめている（どこで開き、何をメンションし、何を頼むか）。

1. 横断ルートでセッションを開き、`START-HERE.md` のキックオフ文をコピペ。`@career-portfolio/CLAUDE.md` ＋ 対象プロジェクトの入力を渡す。
2. AI が `appeals/<会社>/` に下書きを作る（数値・固有名はプレースホルダー）。
3. 実データ（before→after の数値、固有名の伏せ方）を埋めて完成させる。

## 入力ソース（柔軟）

プロジェクトごとに資料の整い方が違うため、入力は柔軟に取る（詳細は `CLAUDE.md`「入力ソースの取り方（柔軟）」）。

- **基本＝ファイルメンション**（タスク資料・PR・チケット）。
- 整理資料が無いプロジェクト＝**git 履歴（コミット/PR）から棚卸し**、**本人メモの貼り付け/口頭**。
- **将来 Slack 連携時＝Slack のやりとりを素材化**。
- 必要なら **`sources/<会社>/` にタスクメモを新規作成**してから整理。

## 運用メモ

- 在職中の準備のため、**社名・パートナー名・人物名・社内固有名はデフォルトで抽象化**する（会社をまたぐため、各社の固有名に注意）。
- **進行中タスクは実名（具体）で書き、完了・レジュメ化時にまとめて抽象化**する（最後の1パス）。実名ドラフトには冒頭注記＋末尾の用語対応表を付ける。詳細は `CLAUDE.md`「固有名の扱い（具体↔抽象のライフサイクル）」。
- 機密・個人情報・未公開の下書きは Git 管理対象外にできるよう、`.gitignore` で `*.local.md` / `private/` を除外している。
- このリポジトリは GitHub の private リポジトリへ接続済み（`origin`）。コミット/プッシュは本人の指示で行う。
