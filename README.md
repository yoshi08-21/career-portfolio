# career-portfolio

業務で得た経験を、転職活動で使える**アピール文（職務経歴書・ポートフォリオ・面接素材）**へ整理・蓄積するための個人リポジトリ。

`task-management` が「目標 → 行動 → 証跡 → 評価」をつなぐのに対し、このリポジトリは **「経験 → 整理 → アピール」** をつなぐことを目的にする。

## 基本の流れ

```text
タスクの関連資料 + 自分の粗いメモ
        ↓ （AI が整理）
アピール文（課題 → 施策 → 工夫 → 成果）
        ↓
職務経歴書・ポートフォリオ・面接で使う
```

## ディレクトリ構成

| パス | 役割 |
|---|---|
| `sources/` | 経験の素材置き場。タスクの箇条書きメモ・関連資料へのリンクなど、整理前のインプット |
| `appeals/` | 完成・下書きのアピール文（成果物）。1テーマ＝1ファイル |
| `templates/` | アピール文テンプレと、AI への依頼テンプレ |
| `resume/` | （任意・必要になったら追加）職務経歴書・履歴書本体 |
| `CLAUDE.md` | このリポジトリで AI に整理を頼むときの方針・入力チェックリスト・出力フォーマット |

## 使い方（1テーマを追加するとき）

1. `sources/` に元メモ・関連資料の場所を残す。
2. `templates/request-template.md` を埋めて AI に依頼する。
3. AI が `appeals/` に下書きを作る（数値・固有名はプレースホルダー）。
4. 実データ（before→after の数値、固有名の伏せ方）を埋めて完成させる。

## 運用メモ

- 在職中の準備のため、**社名・パートナー名・人物名はデフォルトで抽象化**する（「あるパートナー」「業務担当(CS)」等）。
- 機密・個人情報・未公開の下書きは Git 管理対象外にできるよう、`.gitignore` で `*.local.md` / `private/` を除外している。
- GitHub（private リポジトリ）への接続は**次の段階**で行う。現状はローカルのみ。
