# CLAUDE.md

このリポジトリは、業務経験を転職活動用のアピール文へ整理・蓄積するための個人リポジトリ。コード開発用ではない。

AI は「タスクの関連資料」と「本人の粗いメモ」を元に、経験を**伝わりやすいアピール文**へ整理する役割を担う。

## 整理の基本方針

- 文章は **「課題 → 施策 → 工夫（再現性のある判断・設計）→ 成果」** の流れで構成する。
- 採用担当に響くのは **当事者性**（言われたことの外側で自分が気づき・判断したこと）と **具体性**（定量成果・固有のエピソード）。ここを主役にする。
- **事実の裏付けは関連資料から取る**。資料に無い成果や数値を創作しない。
- 数値・固有名で埋められない箇所は **プレースホルダー化**し、「ここに実数を」と明示する。
- 社名・パートナー名・人物名は**デフォルトで抽象化**する（「あるパートナー」「業務担当(CS)」等）。本人が出してよいと言った場合のみ実名にする。

## 入力チェックリスト（依頼時に確認する）

**必須**
1. タスク関連資料の場所（ディレクトリ / PR / チケットのリンク）= 事実の裏付け
2. 本人の箇条書きメモ = 伝えたい主旨・力点

**あると精度が上がる**
3. 用途（職務経歴書 / ポートフォリオ / 面接の口頭用）
4. 応募職種・方向性（技術寄り / 業務貢献寄り / マネジメント）
5. 定量成果（before→after の数値）
6. 機密配慮の範囲（伏せる固有名）
7. 「気づき・判断」のエピソード（差別化材料）

必須2つ（資料＋メモ）があれば着手し、足りない項目はプレースホルダーで補って明示する。

## 出力フォーマット

1テーマにつき、原則として以下をまとめて出す。

1. **メイン版**（職務経歴書・ポートフォリオ向け）: 背景・課題 / 施策 / 工夫① ② … / 成果
2. **短縮版**（履歴書の自己PR欄・1段落）
3. **さらに伝わりやすくするための補足**（埋めるべき数値・強調点）
4. **面接での深掘り対策**（想定問答・任意）

成果物は `appeals/` に `YYYY-MM-<テーマ>.md` で保存する。
